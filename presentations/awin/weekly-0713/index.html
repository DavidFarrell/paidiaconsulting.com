Shift of technical direction

The technical direction is shifting

6 / 16

Where the agent is built

Copilot Studio → LangGraph / Deep Agents
  • Build the real agent in LangGraph / Deep Agents - the same framework Ava runs on
  • Copilot Studio becomes a thin front door to surface the agent to users
    • i.e. a custom agent surfaced through the LiteLLM gateway as an MCP connection to a very basic Copilot agent
7 / 16

Model access and telemetry

  • By going through LiteLLM, we standardise with Awin's approach to building agents
  • We can tie into LangSmith for traces, evals and a continual improvement loop
  • Additional benefits across model and platform flexibility
  • Eventually we can have self-healing benchmark and prompt-rewriting loops, so we can swap models and hill-climb as providers compete
8 / 16

The performance and data layer

  • No longer expecting to build this data layer fully ourselves
  • Avalytics (Awin's BI-plus-AI insights layer) will be one source of insights
  • Semantic data layer (Benni's data platform) is a longer-term home for performance analytics
  • Therefore our data requirements may be smaller than expected
    • However we can't wait for the semantic layer, which may take 18 months or more
    • We can probably access Avalytics quickly
  • We will still need some bespoke analytics calculations and interfaces, which we can build ourselves and may contribute back to the main project
9 / 16

Longer term - Copilot to Ava?

  • I got the impression that over time Ava may become a standard chat interface for staff to access Awin resources
  • If so, we could surface our agent to Ava through the same approach we currently plan for Copilot
10 / 16
