Concerns I've heard - taken seriously

  • "Somebody building everything on-site once again" - the instinctive first reaction to a new parallel build
  • Don't do the same thing twice - separate by user and scope, or plug in rather than rebuild
  • Don't add to teams' workloads - mid-programme asks land on people who already have roadmaps
  • Don't couple too tightly - or AwinOS inherits multi-year timelines it can't afford
  • Adoption is fragile - one unsatisfactory answer loses a user for good (Ava's early lesson)

My stance: light-touch alignment · reuse first · flag anything quick-and-dirty · Wizard-of-Oz before builds · ADR + Security for the durable path
